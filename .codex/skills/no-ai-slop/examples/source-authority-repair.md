# Source-Authority Repair Example

## Weak Artifact

The AEther Flow has proven a complete path from its ontology to general
relativity. The project now shows how gravity emerges from the framework and
provides a robust foundation for future physics.

## Gate Result

`block`

Reason: the copy makes strong scientific claims without source evidence, proof
status, gate status, or limitations. It also promotes the result beyond the
source-authority boundary.

## Repaired Artifact

The AEther Flow website can explain the current source-backed research state,
but it should not present a complete GR derivation unless the upstream project
has registered and reviewed that result. Current public copy should separate
what is reviewed, what is `proposal-only`, what is blocked, and what the next
lawful research step is. When the source state is not inspected in the current
task, use scaffold-level language and link readers to the authoritative source
record instead of strengthening the claim.

## Transferable Lesson

Repair does not mean making weak copy sound more confident. Repair means making
the artifact more truthful, specific, sourced, and useful.
