# Technical Writing Quality Gate Maintenance

## Authority

- Maintain this package under `.codex/skills/technical-writing-quality-gate/`.
- Root `AGENTS.md`, `CONTEXT.md`, live implementation-control records, tracked
  upstream sources, and current validators outrank this maintenance guide.
- Upstream tracked evidence owns scientific, mathematical, governance, and
  research-workflow claims. Website files own presentation behavior only.
- Generated reports, summaries, and memory remain derivative orientation until
  verified.

## Maintenance Rules

- Preserve the explicit `pass`, `repair`, and `block` gate.
- Preserve the distinction between source fact, inference, review signal, and
  validated claim.
- Keep warning-pattern checks advisory by default and describe them as proxy
  evidence only.
- Keep the script Python standard-library-only unless a separately approved
  packet demonstrates a necessary dependency.
- Do not add `.agents` shims, a website `skill.yaml`, package commands, or
  aggregate quality wiring without explicit approval.
- Do not modify `system-analysis` merely to route through this skill. Its native
  source-analysis gate remains independent.
- Do not add public claim wording without inspecting current accepted source
  and review evidence.

## Validation

When package behavior changes, run the help command, focused tests, explicit
Ruff and MyPy checks, full Python tests, implementation-control validation,
aggregate website validation, and `git diff --check`.

When examples or references change, verify that they preserve the upstream
authority boundary, use APA 7 where citations are present, and do not present a
warning scan as truth, source-coverage, originality, or authorship proof.
