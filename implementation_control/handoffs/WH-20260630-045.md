# WH-20260630-045: Greenfield Diagram Rollout

This packet authorizes implementation of the owner-requested Greenfield Diagram
Opportunity Review.

## Completion Summary

Packet `WI-20260630-045` / `WJ-20260630-045-A` added source-backed static
diagram blocks to 28 greenfield routes by reusing already approved public
comprehension PNG assets and their tracked Mermaid dossiers.

Owner feedback on the diagram presentation was also addressed in this packet:
greenfield diagram frames now span the route content band, oversized vertical
diagrams are viewport-contained, every greenfield diagram exposes an in-page
larger-view dialog with direct image fallback, and the repeated visible
post-diagram note was removed from greenfield route blocks.

Owner feedback on caption format was addressed as a bounded refinement:
visible captions in the active greenfield write scope now describe what the
diagram illustrates, while Mermaid source paths remain audit/provenance data
instead of public caption text. Shared figure dialogs and gallery rows no
longer render Mermaid source paths.

Owner feedback on caption weight was addressed as a visual refinement:
diagram description captions and expanded-dialog captions now render as regular
note text rather than bold label text.

Owner feedback on animated page-hero SVG captions was addressed as a bounded
caption-rule refinement: greenfield animated SVG hero captions now state
conservative facts about the route topic instead of describing what the visual
composition illustrates. The public comprehension validator now fails closed if
future inline animated SVG hero figures in the greenfield route scope
reintroduce visual-description caption patterns such as `The diagram
illustrates...`, a visible `Fact:` label, or a lowercase sentence start. A
follow-up refinement removed the visible `Fact:` prefix from those captions and
capitalized each sentence start.

Owner feedback on the `/resources/diagrams/` route was addressed as a bounded
gallery-wiring refinement: the local route now derives its visible inventory
from approved manifest-backed static diagram assets, renders the existing
diagram comprehension block, and groups diagrams by physics, AI workflow,
operations, and source authority concepts. Mermaid source paths remain
manifest/dossier audit data rather than visible card text.

Owner feedback on the `Diagram comprehension` figure was addressed as a shared
renderer rule: comprehension diagrams now default to the full route content
column, the automatic repeated post-diagram note is no longer injected, and
`npm run validate:comprehension` fails closed if the old note, the old
`default` fit option, or the old `940px` wrapper cap returns.

Owner feedback on mobile `Open larger` behavior was addressed as a shared
dialog refinement: expanded diagram dialogs now use a compact close bar, a
scrollable image-first stage, wider-than-viewport mobile diagram rendering, and
caption text below the expanded image.

Owner feedback on the Resources category was addressed as a bounded resource
cleanup refinement: the `/resources/` route map now uses named internal links
in a two-column `Route` and `Reader job` table, dedicated `Provenance and
limits` sections were removed from resource routes, and
`npm run validate:comprehension` now rejects future resource pages that
reintroduce that section or a dedicated Reference/Used-here-for citation table
pattern.

Owner feedback on Resources route-table link contrast was addressed as a
bounded shared-style refinement: `.greenfield-matrix` links now use the
dark-page ivory text color with orange underline, and hover/focus states move
to orange instead of inheriting the dark light-theme cyan link token.

Legacy project-route captions under `src/pages/project/` and the associated
`src/lib/internalExplainers.ts` data still contain older warning-style wording.
Those paths are outside this packet's current authorized write scope and should
be handled by a future implementation-control packet if they are kept in the
public route set.

No upstream source files, route retirements, redirects, dependencies, pushes,
deployments, owner-acceptance claims, or public authority promotions were made.
The lower-priority optional `/license/` split-license decision map was not
created because no existing source-backed static diagram asset matches that
specific concept, and the core route set already received the requested static
diagram coverage.

## Boundary

Allowed work:

- add or reuse static comprehension diagrams for greenfield routes;
- edit tracked Mermaid sources and render PNG assets under
  `public/assets/diagrams/comprehension/`;
- wire diagrams into Astro routes or existing comprehension content modules;
- update asset/source/route/provenance manifests when diagram assets or route
  wiring require it;
- add alt text, captions, provenance, and nearby non-authority boundary copy.
- refine visible diagram captions in the active greenfield route/component
  scope without weakening source-authority boundaries.
- refine animated SVG hero captions in the active greenfield route scope so
  they state conservative page-topic facts instead of visual-composition
  descriptions.

Forbidden work:

- upstream source repository writes;
- source snapshot refresh;
- route retirement or redirect changes;
- dependency additions;
- owner-acceptance claims;
- push or deployment.

## Source-Authority Rule

Diagrams are reader aids. They may clarify relationships, authority flow,
proof burdens, source lineage, lifecycle, and validation boundaries. They do
not create source authority, proof authority, claim promotion, owner
acceptance, deployment readiness, or upstream source changes.
