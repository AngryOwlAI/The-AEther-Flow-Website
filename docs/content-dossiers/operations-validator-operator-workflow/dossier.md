## Glossary

| Term | Plain-language explanation | Boundary note |
| --- | --- | --- |
| Validator | Deterministic repository check. | Not a scientific judge. |
| PASS | A named check accepted the checked state. | Not theorem proof or claim promotion. |
| Receipt | Durable record of command evidence, screenshot evidence, or review status. | Not claim promotion by itself. |
| Screenshot evidence | Rendered public UI evidence. | Still requires human comprehension review. |
| Human gate | Explicit protected decision path for claim promotion or closure. | Cannot be replaced by PASS. |

## Claim boundaries and forbidden implications

### Claim boundaries

- PASS does not prove physics.
- Validation does not grant writes or expand roles.
- Rendered evidence does not replace source authority.
- Sidecar path acceptance is not sidecar adoption.
- Scientific promotion remains separate from operational validation.

### Forbidden implications

- PASS proves a theorem.
- PASS adopts ontology, `M_src`, `g_eff`, matter coupling, Einstein equations, or benchmark status.
- A validator can issue Gate Chair approval.
- Screenshots prove source claims.
- A generated-output validator gives generated pages independent authority.
- A green check resolves unchecked surfaces.
